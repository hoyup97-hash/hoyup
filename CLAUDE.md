# hoyup.kr 프로젝트 가이드

## 공약 페이지 카테고리 구조

이 프로젝트의 공약 페이지는 3개 카테고리로 나뉜다.
카테고리 안에서는 디자인·구조·톤이 반드시 일관되어야 한다.

---

### 1. 주요 4대 공약 `/bus/`, `/pledge-2/`, `/pledge-3/`, `/pledge-4/`
- 대상: 유권자 전체
- 레퍼런스: `/bus/index.html` (가장 먼저 완성된 페이지)
- 테마 컬러: #FF5200 (오렌지)
- 구조 순서: hero → stats-strip → 문제 실태 → 타 사례 → 실행 공약 → slogan
- 새 페이지 추가 시 반드시 bus/index.html을 구조 레퍼런스로 삼을 것

### 2. 대학별 공약 `/ku/`, `/sungshin/`, `/hansung/`
- 대상: 각 대학 재학생
- 레퍼런스: `/ku/index.html`
- 테마 컬러: 학교별로 다름
  - 고려대: #981E32 (크림슨)
  - 성신여대: #6A3FA0 (보라/수정)
  - 한성대: #FF5200 (오렌지, 기본값)
- 공통 스타일: `/assets/style.css` CSS 변수로 오버라이드
- 새 페이지는 `_template_university.html`을 복사해서 시작할 것

### 3. 아파트별 공약 `/apartments/[아파트명]/`
- 대상: 각 아파트 주민 (총 25개)
- 레퍼런스: 첫 번째로 완성된 아파트 페이지 (추후 지정)
- 테마 컬러: #FF5200 (오렌지, 전체 통일)
- 아파트 페이지 추가 시 기존 아파트 페이지와 섹션 구성 동일하게 유지할 것

---

## 일정 수정 규칙

일정 관련 수정 요청이 오면 **반드시 두 파일을 동시에 수정**할 것:
1. `/index.html` — 메인 페이지 일정 카드
2. `/schedule/index.html` — 일정 전용 페이지 (`hoyup.kr/schedule`)

두 파일의 일정 내용은 항상 동일하게 유지해야 한다.

---

## 공통 규칙
- 공통 스타일: `/assets/style.css`
- 폰트: Pretendard (본문), Noto Serif KR (제목)
- 새 카테고리 페이지를 추가할 때는 반드시 해당 카테고리 레퍼런스 페이지를 먼저 읽고 구조를 맞출 것
- CSS를 인라인으로 넣지 말 것. 공통 클래스는 style.css에 추가할 것
